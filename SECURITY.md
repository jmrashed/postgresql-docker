# Security Policy

## Supported versions
This repository supports the latest configurations and scripts in the default branch.

## Reporting a vulnerability
Please do not open public issues for security vulnerabilities.

1. Email the maintainers (preferred) or open a **private** security advisory if your Git hosting provider supports it.
2. Include:
   - what you observed
   - steps to reproduce (if applicable)
   - expected behavior
   - impact assessment

## Response
Maintainters will acknowledge receipt and provide a timeline for investigation and/or release.
